# Tic-Tac-Toe -P- 5.0

## 项目简介
这是一个基于 C 语言开发的经典控制台井字棋（Tic-Tac-Toe）游戏。在本项目中，玩家将与电脑（AI）进行对战。游戏不仅实现了基础的落子与胜负判定功能，还为电脑对手编写了一套基于三种优先级的决策算法，使其具备了一定的博弈能力。

## 核心功能
* **人机对战**：玩家执 `X`，电脑执 `O`。
* **智能 AI 决策**：电脑不会盲目随机落子，而是会根据当前局势进行推演，包含防守、破坏威胁和主动进攻三种策略。
* **状态判定**：游戏能够精准识别胜利（Win）、平局（Draw）和失败（Loose），并实时检测玩家的无效或重复输入。

## 游戏玩法
游戏棋盘由 1 到 9 的数字组成，对应九宫格的各个位置。规则如下：
1. 游戏开始后，由玩家优先落子。
2. 玩家在控制台输入 1-9 之间的数字并回车，即可在对应位置落下 `X`。
3. 电脑会自动思考并在空位落下 `O`。
4. 最先在横线、竖线或对角线上连成 3 个棋子的一方获胜。如果棋盘填满且无人连成 3 子，则为平局。

## 电脑 AI 逻辑说明
游戏中的 AI 包含三个逐步递进的思考层级，通过模拟落子来评估局势：

* **第一层级 (ai_pre_step_one)**：**一击必杀与紧急防守**。AI 会遍历所有空位，如果发现某一步能让自己直接获胜，或发现玩家下一步能直接获胜，则立即抢占该位置。
* **第二层级 (ai_pre_step_two)**：**破坏威胁**。如果当前没有立刻结束比赛的可能，AI 会预测玩家是否会在下一步形成“双杀”（即同时产生两条获胜路线），并提前落子破坏玩家的布局。
* **第三层级 (ai_pre_step_three)**：**主动进攻**。在不需要紧急防守和破坏威胁的情况下，AI 会评估剩余空位，寻找能最大化自己连线概率的位置进行落子。

## 编译与运行
### 环境要求
* 纯 C 语言环境。
* **注意**：本项目使用了 `<conio.h>` 库和 `system("cls")` 命令，建议在 **Windows** 环境下进行编译和运行。在 Linux 或 macOS 系统下可能需要对清屏命令进行修改。

### 编译指令（以 GCC 为例）
打开命令行终端，在代码所在目录下执行：
`gcc main.c -o tictactoe`

### 运行指令
`tictactoe.exe`
